# 使用说明 · 把这个分享包变成你的"AI 学伴"

> 朋友你好——这是一份**完整的"AI 学伴"学习包**，包含了：
> - 一份学习方法论（README.md）
> - 一份可以直接套用的协议模板（CLAUDE.template.md）
> - 一份起点用的补课清单（_backlog.md）
> - 前两节课的精简笔记（notes/）
>
> 把这些落地到你的 Claude Cowork 上，你就能**接着原作者的进度，直接开始第三节课**。

## 落地步骤（5 步，预计 10 分钟）

### Step 1：准备 OpenClaw 项目本体

```bash
git clone https://github.com/openclaw/openclaw.git
cd openclaw
```

如果你已经 clone 过了，直接 `cd` 进去即可。

### Step 2：把分享包内容放进项目

在 `openclaw` 项目根目录下：

1. **把 `CLAUDE.template.md` 复制到项目根目录，重命名为 `CLAUDE.md`**
   ```
   openclaw/CLAUDE.md  ← 协议本体
   ```

2. **把笔记和 backlog 放到 `docs/learning/`**（如果没有这个目录，新建）
   ```
   openclaw/docs/learning/
   ├── _backlog.md
   └── 20260430-项目全景导览.md
   └── 20260430-插件机制解剖.md
   ```
   注意：把 `notes/` 子目录里的两份笔记**直接平铺**到 `docs/learning/` 下，不要保留 `notes/` 这层目录。这是因为协议里 AI 会去 `docs/learning/` 直接找 `YYYYMMDD-*.md` 文件，不会进 `notes/` 子目录。

完成后目录结构应该是：
```
openclaw/
├── CLAUDE.md                                ← 步骤里复制+重命名得到
├── docs/
│   └── learning/
│       ├── _backlog.md
│       ├── 20260430-项目全景导览.md
│       └── 20260430-插件机制解剖.md
├── src/, packages/, extensions/ ...        ← OpenClaw 项目本体
└── ...
```

### Step 3：填写 CLAUDE.md 里的三处占位符

打开刚才放进去的 `CLAUDE.md`，找到三处用 `<>` 包裹的占位符：

| 占位符 | 改成什么 |
|---|---|
| `<语言>` | 你想用什么语言交流（例：中文 / English） |
| `<项目名>` | `openclaw`（已经定好了） |
| `<用户背景>` | **这是最关键的一项**——写你的编程经验、强项、弱项、学习目标 |

**重要：用户背景越具体，AI 越能精准对你说话**。

参考写法：
> - Java 入门级，写过几个小项目；几乎不熟 TypeScript
> - 熟悉后端逻辑设计，但对前端构建工具链（Vite、tsdown 等）一片空白
> - 学习目标：通过精读 OpenClaw 系统性补齐"现代 TS/Node 工程"这一整块知识

不要客气、不要谦虚——AI 越知道你现在在哪，越能选对节奏和深度。

### Step 4：按你自己水平更新 `_backlog.md`

打开 `docs/learning/_backlog.md`——它现在的状态（⬜/🔄/✅）是分享版的**起点状态**。

**花 2-3 分钟把它改成你自己的状态**：
- 你已经熟悉的概念 → 打 ✅
- 你半懂的 → 打 🔄
- 你完全没概念的 → 留 ⬜

例：你如果是 Java 老手第一次接触 Node，那 `Node.js 与 npm 生态` 就是 ⬜，但 `控制反转 (IoC)` 可能因为 Spring 经验是 ✅。

这一步会让接下来的学习节奏**精准贴合你的水平**。

### Step 5：在 Claude Cowork 打开项目并开口

1. 打开 Claude（Cowork mode），**让它选择 `openclaw` 这个文件夹作为工作目录**。
2. 直接发一句：

   > **"开始下一节课。"**

   或者：

   > **"我已经看过前两节课的笔记，今天想从 src/entry.ts 跟启动流程。"**

   AI 会自动：
   - 读 `CLAUDE.md`（学习协议）
   - 列出 `docs/learning/` 下的笔记
   - 读最新一份（插件机制解剖）了解上次结尾的方向
   - 对你说一句类似"上次我们完成了【插件机制解剖】，最后讨论了 IoC、Provider 模式。今天想从哪里开始？"
   - 你回复方向后，正式开课

**如果 AI 没自动这么做**——比如它直接反问"我们之前在做什么？"——那是协议没生效。检查 `CLAUDE.md` 是否在项目根目录、内容是否完整。

## 心理准备 · 这套方法不一样的地方

读完 `README.md` 你会知道方法论。这里再补几条**实操心态**：

1. **AI 会抛问题让你猜**——别怕答错，**答错才是最大收获**。猜的过程已经在调用知识，听一次解释就是巩固而不是初学。
2. **不要嫌"先全景后局部"慢**——前 1-2 节课像在"低头铺路"，但跳过它一定迷路。
3. **每节课结束 AI 会写笔记**，下次会话只需说"继续"——AI 会自己读笔记接上进度。**不要重新交代上次到哪**——那是协议失败的迹象，应该让你来纠正它。
4. **节奏不合立刻反馈**——觉得太快说"等等"，觉得太慢说"再深入一点"，协议是活的。
5. **当 AI 提示"建议下次新开会话"**——照做。一节课闭环时换会话比硬撑下去体验稳定得多。

## 接续到第三节课

原作者刚开始第三节课（**主程序启动流程**——从 `package.json` 的 `bin` 字段一路追到 `src/entry.ts`）。

如果你想**精确同步**到这个进度，第一句话可以直接说：

> "今天想从主程序启动流程开始——package.json 的 bin → openclaw.mjs → dist/entry.js → src/entry.ts 这条链路。"

AI 会按协议先做"承上启下"——把上节课的"插件机制"和这节课的"启动流程"串起来，再往前推进。
