# 把 AI 变成你的"学伴"——一种系统性研读开源项目的方法

> 这不是"让 AI 帮我读代码"的教程。
> 这是"和 AI 一起当学伴、一节一节扎实地学"的协议。

## 你想解决的问题

你想通过研读优秀开源项目来系统性提升技术能力，但常陷入两类困境：

- **自己读**：项目动辄上百个文件，看着 README 不知从哪下手，进入代码两天就放弃
- **让 AI 帮我读**：直接让 AI 讲解，听的时候很懂，关掉对话三天就忘干净

这份方法解决的是这两者之间的路：**让 AI 当持续陪伴的学伴**——像师傅带徒弟一样，一节课一节课带你进去；而不是一次性把答案塞给你。

## 核心理念：从「答案机」到「学伴」

|  | 答案机模式 | 学伴模式 |
|---|---|---|
| 互动 | 你问 AI 答 | AI 也会抛问题让你猜，再对照代码验证 |
| 节奏 | 任意，常一发不可收 | 中等节奏，一节课覆盖一个小模块 |
| 深度 | 偏快偏全 | 偏深偏稳，宁可少讲要讲透 |
| 留痕 | 关掉对话什么都不剩 | 每节课结束写笔记，下次自动接续 |
| 盲区 | 一带而过 | 标记 + 集中补课，绝不丢失 |

## 六维度协议（最小可行版本）

| 维度 | 协议 |
|---|---|
| **路径** | 先全景（1-2 节课），再下沉到局部模块 |
| **互动** | 事实类直接讲；判断类先让你猜，再对照代码验证 |
| **节奏** | 一节课一个小模块（一个类/一个文件/一组函数），宁少勿滥 |
| **动手** | 复述、画图、改一行代码看运行结果——每个模块至少一次 |
| **笔记** | 每节课结束 AI 主动写笔记到 `docs/learning/`，下次会话自动读 |
| **盲区** | 不打断主线；遇到不熟的概念登记到 `_backlog.md`，集中补课 |

## 3 步上手

### Step 1：选一个想学的开源项目
在 GitHub 上找一个你"想学透"的项目，`git clone` 到本地。不一定要大，但要"有营养"。

### Step 2：复制协议模板
把本目录下的 `CLAUDE.template.md` 复制到你的项目根目录，重命名为 `CLAUDE.md`。

打开它，改三处占位符（用 `<>` 包裹的部分）：
- `<语言>`：你想用什么语言交流（中文 / English / ……）
- `<项目名>`：你的项目名
- `<用户背景>`：你的编程经验、强项弱项、学习目标——越具体，AI 越能精准对你说话

### Step 3：开第一节课
打开 Claude（任何能读项目目录的版本——Claude Code、Cowork mode 等），直接说一句：

> "开始第一节课。"

之后 AI 会按协议节奏带你走。每次新会话只需说"继续"或"开始下一节课"，AI 会自动读笔记接上进度。

## 一段真实片段（来自一次实战学习）

> **AI**："如果要让这个项目支持一个新的 AI 模型，你觉得最有可能改哪个目录？A/B/C/D"
>
> **学习者**："C。理由是这个项目用插件化设计，新 AI 等于新插件。"
>
> **AI**："✅ 答案 C 正确。证据：`extensions/anthropic/` 是独立 npm 包……顺便指出你回答里把『插件』和『skill』混用了，这是两个不同粒度的概念……"

这种"先猜后验"的循环，比直接告诉答案更让人记得住——**你的脑子已经先调用过这部分知识，再听一次解释就是巩固而不是初学**。

## 一些"小事"

- 不要嫌"先全景后局部"慢——跳过它一定迷路
- 不要嫌"写笔记"麻烦——笔记是 AI 写的，你只看一眼
- 不要嫌 AI 抛问题——答错才是最大收获
- 节奏不合立刻反馈——协议是活的，可以按你的吸收能力随时调整

## 想看一段这种学习方式实际产出的笔记长啥样？

去 `notes/` 目录下翻几篇——那是真实学习对话浓缩出来的"分享版精简笔记"，去掉了私人化细节、保留了方法论闪光点。看完应该能体感到这种方式的味道。
